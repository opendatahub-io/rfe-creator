## Summary

AI Engineers using gen AI studio need full prompt template management capabilities through the existing MLFlow prompt integration. Today, users can only load and save simple system prompts, but MLFlow supports parameterized text templates with `{{variable}}` placeholders and multi-role chat templates with multiple turns. The current limitation forces users to leave gen AI studio and work directly in the MLFlow UI or use competing tools like wx.ai PromptLab, breaking their workflow within RHOAI. Credit Alliance has explicitly conditioned their RHOAI migration on this capability, and multiple existing prompt lab customers are waiting for feature parity before migrating.

## Problem Statement

AI Engineers building generative AI applications within RHOAI cannot work with parameterized text prompt templates or multi-role chat prompt templates from gen AI studio. Specifically, they cannot:

- Browse or search the MLFlow prompt registry for text and chat template types
- Load templates with `{{variable}}` placeholders and view or fill in those variables
- Run inference with substituted variable values
- Create new text or chat prompt templates and save them back to the MLFlow registry as versioned entries

As a result, they must switch to the MLFlow UI directly or use competing tools like wx.ai PromptLab, which already offers this end-to-end prompt management experience. This breaks their workflow within RHOAI and is a documented reason customers choose competing platforms.

## Affected Customers

- **Credit Alliance** -- named customer commitment; migration to RHOAI is explicitly conditioned on this capability being available
- **Multiple existing prompt lab customers** -- waiting on RHOAI feature parity with wx.ai PromptLab before migrating
- **AI Engineers** building generative AI applications within RHOAI who need prompt template management integrated into their existing workflow

## Business Justification

- **Customer retention and migration risk**: Credit Alliance has explicitly conditioned their RHOAI adoption on this capability, making it a direct blocker for their migration and associated revenue
- **Competitive positioning**: wx.ai PromptLab currently offers this end-to-end prompt management experience, and the lack of parity is a documented reason customers choose competing platforms over RHOAI
- **Broad migration enabler**: Multiple existing prompt lab customers are waiting on this feature before migrating to RHOAI, making it a gating capability for a customer segment, not just a single account
- **Strategic investment alignment**: Gen AI studio is a strategic investment area and needs full MLFlow prompt management to be a credible alternative in the market

## Acceptance Criteria

- [ ] Users can browse and search the MLFlow prompt registry for all template types (text and chat) from gen AI studio
- [ ] Users can load a parameterized text prompt template, view its `{{variable}}` placeholders, fill in values, and run inference with the substituted prompt
- [ ] Users can load a multi-role chat prompt template, view its turns and variables, fill in values, and run inference
- [ ] Users can create new text or chat prompt templates and save them as versioned entries in the MLFlow prompt registry
- [ ] Users can attach registered MLFlow models to prompt template sessions for inference
- [ ] Existing system prompt loading behavior from RHOAI 3.4 is preserved and continues to function

## Success Criteria

- Credit Alliance's stated prerequisite for RHOAI adoption is met, unblocking their migration
- Feature parity with wx.ai PromptLab prompt management capabilities is achieved for text and chat template types
- AI Engineers can complete their full prompt template workflow (browse, load, edit, test, save) without leaving gen AI studio

## Scope

### In Scope
- Extending the existing MLFlow prompt integration in gen AI studio to cover text and chat template types
- Variable management (viewing, filling, substituting) for parameterized templates
- Browsing, loading, creating, and saving prompt templates in the MLFlow registry
- Preserving existing system prompt loading behavior

### Out of Scope
- Changes to MLFlow internals
- Prompt evaluation or batch testing pipelines
- Prompt optimization (e.g., DSPy-based approaches)
- Capabilities not supported by MLFlow's prompt registry
