## Summary

OpenShift AI notebook images must be validated and certified for NVIDIA's Vera Rubin architecture (NVL72) at launch, ensuring data scientists can use GPU-accelerated notebooks on Vera Rubin hardware without manual environment configuration. This includes validating the existing PyTorch/CUDA notebook image on Vera Rubin silicon, providing ARM (aarch64) notebook image variants for the Vera CPU, and ensuring driver and toolkit alignment with NVIDIA's Vera Rubin software stack. Readiness is required for Tech Preview by June 2026 and GA by October 2026, aligned with NVIDIA's Vera Rubin launch milestones.

## Problem Statement

Today, the v2025.1 "Jupyter | PyTorch | CUDA | Python 3.12" notebook image works with CUDA 13.0 and detects GPUs, but Vera Rubin GPUs are not formally validated, ARM (aarch64) notebook images for the Vera CPU are not available, and driver/toolkit alignment with NVIDIA's Vera Rubin software stack is not guaranteed. Data scientists deploying on Vera Rubin NVL72 hardware cannot be confident that notebooks will work correctly without manual environment setup. There is no multi-arch support for the ARM-based Vera CPU, meaning teams with Vera Rubin infrastructure have no supported notebook path for the CPU component of the platform.

## Affected Customers

- Enterprise organizations deploying OpenShift for NVIDIA infrastructure on latest NVIDIA hardware
- Neo-cloud providers and sovereign cloud operators adopting Vera Rubin NVL72 platforms
- Data science teams using OpenShift AI notebooks for AI/ML development on NVIDIA GPUs
- NVIDIA (key partner) -- Day 0 notebook readiness is required for the OpenShift for NVIDIA partnership stack

## Business Justification

Day 0 GPU support in notebooks is table stakes for the OpenShift for NVIDIA partnership stack. Notebook images are the primary entry point for data scientists on OpenShift AI -- without Vera Rubin readiness at launch, the entire developer experience layer of the stack is incomplete for organizations deploying on next-generation NVIDIA hardware. This is a strategic investment directly aligned with NVIDIA Vera Rubin milestones (Tech Preview June 2026, GA October 2026) and supports the OpenShift for NVIDIA fast lane program. Failure to deliver notebook readiness at Vera Rubin launch would leave the OpenShift for NVIDIA developer experience incomplete at a critical partner milestone.

## Acceptance Criteria

- [ ] The "Jupyter | PyTorch | CUDA | Python 3.12" notebook image launches successfully on OpenShift for NVIDIA with Vera Rubin NVL72 hardware, with GPU detection and utilization working correctly (torch.cuda.is_available() returns True, torch.cuda.get_device_name() identifies the Vera Rubin GPU)
- [ ] PyTorch training and inference workloads execute successfully on Vera Rubin GPUs from within notebooks, including multi-GPU scenarios
- [ ] An ARM aarch64 notebook image variant is available and functional on the Vera CPU
- [ ] NVIDIA driver version alignment meets minimum requirements specified in Vera Rubin release notes
- [ ] The notebook image is available in the OpenShift AI image catalog at Vera Rubin launch dates (Tech Preview June 2026, GA October 2026)
- [ ] Documentation covers Vera Rubin support details, driver requirements, CUDA version information, and migration guidance from Grace Blackwell (CUDA 12.x) to Vera Rubin (CUDA 13.x)

## Success Criteria

Data scientists can launch a supported OpenShift AI notebook on Vera Rubin NVL72 infrastructure and immediately begin GPU-accelerated PyTorch development without manual environment configuration. ARM-based Vera CPU users have a supported notebook image variant. The notebook experience on Vera Rubin hardware is equivalent to the existing experience on prior NVIDIA architectures, available from Day 0 of each Vera Rubin milestone.

## Scope

### In Scope
- CUDA 13.x validation on Vera Rubin NVL72
- PyTorch with Vera Rubin GPU kernel support validation
- ARM aarch64 multi-arch notebook image builds
- NVIDIA driver and toolkit alignment for Vera Rubin
- GPU detection and multi-GPU support validation
- Base OS alignment with RHEL for NVIDIA baseline
- Documentation and migration guidance

### Out of Scope
- CUDA 14.x support
- Non-notebook components (inference server, llm-d)
- x86 Rubin NVL8
- Custom user-built images
- Non-OpenShift for NVIDIA platforms
- TensorRT/cuDNN updates (stretch goal, not MVP)
- Additional notebook image variants such as TensorFlow (stretch goal, not MVP)
