## Summary

Enterprise customers building agentic AI workflows on RHOAI need multi-turn tool calling through Llama Stack's Responses API to work reliably across conversation turns. Today, tool calling degrades progressively as conversations grow: reasoning context is lost between turns and tool call parsing quality varies across models. This blocks major accounts from adopting RHOAI's agentic platform capabilities and puts deal expansion at risk.

## Problem Statement

Two compounding issues make multi-turn tool calling through Llama Stack unreliable for production agentic workflows:

1. **Reasoning context lost between turns.** Llama Stack's Responses API is built on chat completions, which causes reasoning text to be stripped between turns. When a model reasons about tool results and then needs to make a subsequent tool call, the reasoning from prior turns is unavailable. Each successive turn operates with less context, leading to increasingly incoherent tool use decisions.

2. **Inconsistent tool call parsing across models.** vLLM's tool call parsing quality varies significantly across supported models. Per-model parsers are inconsistently developed and tested, so the same multi-turn scenario may work on one model but fail on another.

These issues compound: small parsing inconsistencies accumulate across turns while diminishing reasoning context makes the model less able to recover from errors. The result is progressive reliability degradation -- tool calling may work for 2-3 turns but becomes unusable in longer agentic conversations. Customers are forced to work around this by calling vLLM directly via Chat Completions, bypassing Llama Stack entirely and losing the orchestration capabilities the Responses API provides.

## Affected Customers

- **Altus Capital** -- Major enterprise account building internal AI tooling on RHOAI. Their agentic workflows require multi-turn tool calling for complex, multi-step tasks. Regulated financial institution requiring self-hosted infrastructure.
- **Meridian Bank** -- Major enterprise account running agentic workflows in regulated financial services on RHOAI. Also requires self-hosted infrastructure due to regulatory constraints.

Both accounts are evaluating hosted alternatives (e.g., OpenAI's integrated tool calling) as a path of least resistance if RHOAI's agentic capabilities remain unreliable.

## Business Justification

Altus Capital and Meridian Bank represent major enterprise accounts whose agentic AI initiatives depend on reliable multi-turn tool calling. Deal expansion is at risk if these accounts cannot build production-grade agent workflows on RHOAI. Both are regulated financial institutions that require self-hosted infrastructure, making them ideal RHOAI customers -- but hosted API providers with proven tool calling (such as OpenAI's Responses API) become competitive alternatives if RHOAI's offering does not work reliably. Llama Stack is RHOAI's strategic answer to hosted agentic APIs; if multi-turn tool calling through it is unreliable, the entire agentic platform positioning stalls.

## Acceptance Criteria

- [ ] A multi-turn tool calling scenario (model calls a tool, receives the result, reasons about it, then calls another tool) completes successfully through Llama Stack's Responses API without degradation across turns
- [ ] Reasoning text generated by the model is preserved across turns so that subsequent tool calls benefit from prior reasoning context
- [ ] Tool call parsing quality when calling models through Llama Stack matches the quality observed when calling the same model and prompt directly via vLLM Chat Completions
- [ ] Multi-turn tool calling is validated end-to-end across multiple supported models, both directly against vLLM and through Llama Stack
- [ ] Customers can build multi-turn agent workflows on the Responses API without needing to fall back to direct vLLM access as a workaround

## Success Criteria

Multi-turn agentic workflows through Llama Stack's Responses API are reliable enough for production use by regulated enterprise customers. Specifically: tool calling accuracy does not degrade as conversation length increases, and customers like Altus Capital and Meridian Bank can build their agentic applications entirely through the Responses API without bypassing it for direct vLLM access.
