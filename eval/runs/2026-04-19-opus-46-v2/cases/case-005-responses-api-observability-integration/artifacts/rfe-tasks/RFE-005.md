## Summary

Enterprise customers running AI workloads on RHOAI have no visibility into Responses API behavior in production. Operations teams cannot trace requests, measure token consumption, or monitor latency and error rates for Responses API calls, making it impossible to debug production issues, optimize inference costs, or meet SLA requirements. Customers need GenAI-specific observability instrumentation for the Responses API, built on OpenTelemetry with GenAI semantic conventions, so that every API call is traceable and measurable.

## Problem Statement

When enterprises deploy AI agents and LLM inferencing pipelines on RHOAI using the Responses API, operations teams are completely blind to what is happening at the API level. There is no telemetry emitted for Responses API calls -- no trace spans, no token usage metrics, no latency or error rate data. This means:

- Operations teams cannot trace a request end-to-end through the agent-to-LLM pipeline to diagnose failures or performance degradation.
- Finance and platform teams cannot measure token consumption for cost attribution, capacity planning, or chargeback to individual users or agent workloads.
- SRE teams cannot set up dashboards or alerts for Responses API latency, throughput, or error rates, making SLA compliance unverifiable.
- Data science teams cannot correlate production inference behavior with experiment tracking to understand how model changes affect real-world performance.

There are no viable workarounds -- customers cannot instrument the Responses API externally because the telemetry must be emitted from the serving layer itself.

## Affected Customers

- **Orion Travel Systems** -- 400+ AI initiatives; explicitly stated "OpenTelemetry is really important -- to trace between agent and LLM." Production readiness is blocked without this capability.
- **Nova Wireless** -- Needs comprehensive observability for AI agents including token usage and performance metrics.
- **Keystone Financial** -- Requires comprehensive observability and monitoring of the entire LLM inferencing pipeline for compliance and operational confidence.
- **Posta Nazionale** -- Needs real-time model performance visibility across operations.

All four are large enterprise accounts with significant AI deployments and expansion potential. Orion Travel Systems alone represents a strategic account with 400+ AI initiatives at risk of stalling without production-grade observability.

## Business Justification

Four named enterprise customers with large-scale AI deployments are blocked or at risk of stalling their RHOAI adoption because they cannot operate Responses API workloads in production without observability. Without instrumentation, these customers cannot move from pilot to production -- a critical adoption milestone. Orion Travel Systems (400+ AI initiatives), Nova Wireless, Keystone Financial, and Posta Nazionale all independently cite observability as a requirement for production readiness and SLA compliance. These are strategic accounts with significant expansion potential that depends on production deployment success.

## Acceptance Criteria

- [ ] Every Responses API call emits a traceable OpenTelemetry span with GenAI semantic convention attributes (including token usage metrics such as input tokens and output tokens, response model identifier, and operation metadata).
- [ ] Operations teams can view end-to-end distributed traces spanning from agent calls through to LLM inference for Responses API workloads.
- [ ] A monitoring dashboard provides real-time visibility into Responses API latency, token usage, throughput, and error rates.
- [ ] Per-user and per-agent usage attribution is available, enabling platform teams to perform cost allocation and chargeback for Responses API consumption.
- [ ] Responses API telemetry integrates with experiment tracking so that data science teams can correlate production inference behavior with model experiments.

## Success Criteria

- Every Responses API call generates a traceable OTEL span with GenAI attributes (e.g., `gen_ai.usage.input_tokens`, `gen_ai.response.model`).
- A dashboard shows real-time latency, token usage, and error rates for Responses API workloads without additional customer instrumentation effort.
- End-to-end trace visibility is available through OpenTelemetry integration, enabling operations teams to trace requests from agent to LLM and back.
- Platform administrators can attribute usage to specific users or agents for chargeback purposes.
