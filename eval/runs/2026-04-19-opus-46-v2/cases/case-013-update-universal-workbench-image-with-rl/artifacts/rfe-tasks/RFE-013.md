## Summary

RHOAI enterprise customers and internal AI/ML platform teams who use Training Hub for fine-tuning and alignment workflows need the universal workbench image to include all dependencies required for RLVR (Reinforcement Learning from Verifiable Rewards) training. Today, users must manually install these dependencies into their workbench environments before they can run LoRA adapter-based RLVR workflows. This setup friction wastes time, causes version inconsistencies across deployments, and results in example RLVR notebooks that fail out-of-the-box — undermining confidence in the platform.

## Problem Statement

Users who want to develop and experiment with LoRA adapter-based RLVR training in RHOAI must currently perform manual dependency installation in their workbench environments before they can begin any work. This creates several problems:

- **Setup friction**: Every new workbench session requires manual installation steps, wasting time that should be spent on model alignment work.
- **Version inconsistencies**: Different users and deployments end up with different dependency versions, leading to reproducibility issues and hard-to-diagnose failures.
- **Poor first experience**: Example RLVR notebooks provided with Training Hub fail to run without additional manual installation, creating a poor first impression of the platform's RLVR capabilities.
- **Low discoverability**: Because RLVR dependencies are not present by default, users may not realize the platform supports these workflows at all.

## Affected Customers

- RHOAI enterprise customers using Training Hub 0.5+ for fine-tuning and alignment workflows
- Internal AI/ML platform teams adopting RLVR for model alignment
- Partners integrating with OpenShift AI for LLM training pipelines

## Business Justification

RLVR is a key emerging capability for LLM alignment, and Training Hub 0.5+ is exposing it as a supported workflow. Without pre-packaged dependencies in the workbench image, adoption is hampered by avoidable setup friction. Example notebooks that fail out-of-the-box create a poor first impression that undermines customer confidence in the platform's readiness for production RLVR workflows. Ensuring seamless RLVR support strengthens RHOAI's competitive positioning in the enterprise AI training market, where ease-of-use and time-to-first-experiment are key differentiators.

## Acceptance Criteria

- [ ] Users can run RLVR training workflows immediately on a fresh universal workbench without any manual dependency installation
- [ ] Example RLVR notebooks execute successfully out-of-the-box on a new workbench instance
- [ ] RLVR dependencies function correctly across CUDA, ROCm, and CPU backends
- [ ] No dependency conflicts are introduced with existing Training Hub 0.5+ and Kubeflow SDK dependencies
- [ ] All required RLVR dependencies are published to AIPCC indexes
- [ ] Documentation is updated to reflect the included RLVR capabilities and supported workflows

## Success Criteria

- Zero manual installation steps required for users to begin RLVR training on a fresh workbench
- No dependency conflict reports from users after the workbench image update
- Example RLVR notebooks pass automated validation across all supported backends (CUDA, ROCm, CPU)
