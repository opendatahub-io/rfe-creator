## Summary

Platform Operators deploying the Red Hat AI portfolio need the operator to carry a name that reflects the full breadth of the portfolio it manages. The current "Red Hat OpenShift AI" (RHOAI) name implies the operator is specific to OpenShift, creating confusion as the portfolio expands to include products like Red Hat AI Inference Server (RHAIIS) that span both xKS and OpenShift. Renaming to "Red Hat AI" (RHAI) Operator establishes a coherent product identity that scales with the portfolio and avoids the need to create a separate operator for each new AI product on OpenShift.

## Problem Statement

Platform Operators encounter an operator named "Red Hat OpenShift AI" and reasonably conclude it is scoped to OpenShift-specific AI capabilities. As Red Hat expands its AI portfolio to include products like RHAIIS with distributed inference across xKS and OpenShift, this naming mismatch creates two problems:

1. **Customer confusion**: The operator name does not match the breadth of products it manages, undermining confidence in the product identity and creating unnecessary onboarding friction for teams adopting new portfolio products.
2. **Portfolio scalability**: The current name blocks natural reuse of the existing operator for new products. Without a rename, Red Hat would need to build and maintain a separate operator for each new AI product on OpenShift, increasing development cost and support burden with no customer benefit.

There is no workaround that addresses the root cause -- no amount of documentation can compensate for a product name that contradicts the product's actual scope.

## Affected Customers

- Platform Operators deploying Red Hat AI portfolio products (RHOAI, RHAIIS, and future offerings) who see inconsistent product identity across the portfolio
- Enterprises currently running RHOAI on OpenShift who plan to adopt RHAIIS and need a coherent operator experience across products
- Red Hat partners integrating with the AI operator ecosystem who need stable, portfolio-aligned naming for their tooling and documentation

## Business Justification

The Red Hat AI portfolio is expanding beyond OpenShift-specific tooling. A portfolio-aligned operator name is a prerequisite for coherent product identity as new products launch. Specifically:

- **Eliminates duplicate engineering investment**: Without the rename, each new AI product on OpenShift would require a separate operator, duplicating development and ongoing support costs. The rename enables reuse of the existing operator infrastructure for RHAIIS and future products.
- **Reduces onboarding friction**: Platform Operators deploying multiple Red Hat AI products will encounter a single, consistently named operator rather than a confusing mix of product-specific and platform-specific naming.
- **Strategic portfolio alignment**: As a 2026 strategic investment, the Red Hat AI portfolio expansion requires naming that scales. The current RHOAI name is an active impediment to launching RHAIIS on OpenShift under a coherent brand.

## Acceptance Criteria

- [ ] All customer-facing operator artifacts (CRDs, API groups, container images, namespace defaults, CLI references) carry the RHAI name with no remnants of RHOAI naming
- [ ] Existing RHOAI installations can upgrade to RHAI without redeploying managed workloads
- [ ] Custom Resources created under RHOAI continue to reconcile after upgrade without manual migration by the Platform Operator
- [ ] Metrics, distributed traces, and telemetry emitted by the operator reflect the new RHAI naming convention
- [ ] Platform Operators can confirm rename completion and operator health using existing monitoring tooling

## Success Criteria

- Platform Operators installing or upgrading the operator see "Red Hat AI" consistently across all touchpoints, with no ambiguity about whether the operator supports products beyond OpenShift AI
- Existing RHOAI customers experience a seamless upgrade path with zero disruption to running workloads -- no manual CR migration, no redeployment of managed services
- Installation guide, upgrade guide, and CRD reference documentation are updated to reflect the new name with explicit transition steps for existing RHOAI users
